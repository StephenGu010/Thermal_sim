# Thermal_sim 工程说明

本文档给后续开发者和 AI agent 快速说明当前工程的输入模式、处理链路和调试重点。

## 1. 项目定位

`Thermal_sim` 是 Tiny1-C 热成像瞄具效果的 PC 端仿真工具。它用 PySide6 + OpenCV 读取 Mock/Tiny1-C/UVC 输入，输出带 HUD 的 16:9 瞄具画面，并保存最终截图。

当前目标是毕业设计展示和算法调试，不是复现厂商专有算法，也不输出真实摄氏温度。

## 2. 输入与 Profile

顶部工具栏有两个关键选择：

- `Source`：`Mock` 或 `UVC x`。
- `Profile`：
  - `Thermal Tiny1-C`：主模式。用于 Mock 热图或 Tiny1-C。优先使用 Y16/raw14，8-bit UVC 仅作为 fallback。
  - `Visible Demo`：普通可见光摄像头演示模式。只做黑底亮边风格，不代表热成像，也不做热目标分类。
- `Resolution`：
  - `Native 256x192`：按 Tiny1-C 原生热像尺寸采集/处理。
  - `HD 640x480`、`720p 1280x720`：UVC 优先请求更高分辨率，实际是否成功取决于设备。
  - `Tiny x2 Detail`：采集仍为 256×192，Outline 内部上采样到约 2 倍做边缘处理。
- `Detail`：`Clean/Balanced/Fine/Ultra`，控制边缘密度、阈值、补线强度和内部处理倍率。
- `Smooth`：`Off/Low/Mid/High`，控制时域 EMA、Gaussian/bilateral 低通和轻微 glow。

状态栏会显示 UVC 打开状态，例如 `mode=Y16(raw)` 或 `mode=8bit(fallback)`，并显示请求/实际分辨率。HUD 左下角会显示 `THERMAL RAW 256x192 x2` 或 `VISIBLE DEMO 1280x720`。

## 3. 核心数据流

Thermal 主链路：

```text
Tiny1-C/Mock raw14
→ AGC/DDE-like WHOT/BHOT 增强
→ thermal target mask
→ contour extraction
→ rule-based person/object candidate
→ scope_renderer 输出最终 HUD 帧
```

Thermal Outline 链路：

```text
raw14
→ bad-pixel suppression
→ temporal EMA
→ optional internal upsample
→ light spatial denoise
→ thermal target gate
→ Sobel/Scharr
→ NMS + hysteresis
→ edge density cap
→ gap bridge
→ hard-edge outline mapping
```

Visible Demo 链路：

```text
visible BGR/gray
→ grayscale
→ optional high-resolution capture / internal upsample
→ bilateral/Gaussian denoise
→ mild contrast stretch
→ Sobel/Scharr + Canny-like linking
→ small component cleanup
→ edge density cap
→ visible demo outline
```

## 4. 主要文件

- `core/camera_capture.py`：Mock/UVC 采集，统一输出 `raw14`，并把原始 UVC 帧传给 UI。
- `core/outline_processing.py`：双 Profile Outline 算法，包含内部上采样、硬边映射、thermal gating、时域 EMA、可见光演示链路。
- `core/scope_renderer.py`：把 WHOT/BHOT/Outline 和 HUD 合成为最终 960x540 BGR 帧。
- `ui/main_window.py`：Profile/Resolution/Detail/Smooth 选择、采集生命周期、帧处理编排和截图保存。
- `docs/holosun_outline_reverse_engineering.md`：Outline 逆向假设、边界和参数说明。

人工验收：

1. `Mock Person + Thermal Tiny1-C + OUTLINE ON`：主体轮廓连续，背景尽量压黑。
2. `Mock Person + Thermal Tiny1-C + Tiny x2 Detail + Fine + Low Smooth + OUTLINE ON`：边缘应比 Native 更细、更硬。
3. `UVC + Visible Demo + 640x480/720p + OUTLINE ON`：普通摄像头呈黑底亮边，但 HUD 明确显示 `VISIBLE DEMO`。
4. Tiny1-C 连接时确认状态栏显示 `Y16(raw)`；若显示 `8bit(fallback)`，说明效果会退化。

## 6. 调参建议

- 想减少杂边：提高 `high_percentile`，降低 `thermal_edge_density` 或 `visible_edge_density`。
- 想让人体轮廓更连续：适度提高 `bridge_max_gap` 或降低 `low_ratio`。
- 可见光模式纹理太多：降低 `visible_edge_density`，或提高 `visible_min_component_area`。
- 边缘太糊：优先使用 `Tiny x2 Detail + Fine + Low Smooth`，并避免 `High Smooth`。
- 线条灰度梯度太明显：提高 `edge_hardness` 或关闭 glow。
- Tiny1-C 实测优先保证 Y16/raw 输入；不要从伪彩色 BGR 图再做 Sobel。
